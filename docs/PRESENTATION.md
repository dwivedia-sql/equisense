# EquiSense — 7-Minute Presentation Script

**Total time:** 7 minutes + 3 minutes Q&A
**Format:** Live demo on laptop. Wear headphones for the hook. Bring a spare pair for judges.
**Print this doc and hold it during rehearsals until you no longer need it.**

## 0:00 – 0:30 | Hook (Aarav)

*[Put on headphones. Open a screen reader. Navigate to a page with the quadratic formula.]*

Play the screen reader audio first — without saying anything. Let the garbled output speak for itself.

Then switch to EquiSense. Navigate the same formula with arrow keys. Let the HRTF spatial audio play. Numerator above, denominator below, square root pulling closer.

Still say nothing until the audio contrast lands.

Then: **"That's the difference between a screen reader and EquiSense."**

---

## 0:30 – 1:30 | Problem & Solution (Aarav)

> "The National Federation of the Blind estimates that fewer than 10% of blind students pursue STEM degrees. A major reason is that the tools they rely on — screen readers — are completely blind to mathematical structure. A screen reader reads the quadratic formula as: x equals open paren minus b plus or minus square root open paren b squared minus four a c close paren close paren divided by two a. It's flat. It's linear. There's no hierarchy, no structure."

> "EquiSense solves this with three features: an equation navigator with true 3D spatial audio, a graph sonifier that announces patterns before playing them, and a camera that captures printed equations and makes them immediately navigable. Let me show you."

## 4:30 – 5:30 | Architecture (Person 3)

*[Show the architecture diagram from DESIGN.md or a printed version.]*

**Person 3:** "Three independent modules share one resource: a Web Audio AudioContext. The equation navigator uses HRTF PannerNodes — that's the spatial audio standard built into every modern browser. The graph sonifier schedules frequency ramps using AudioContext.currentTime, not setTimeout, so even on battery power the sync never drifts."

> "The math-aware OCR layer is what separates this from slapping Tesseract on a camera. Tesseract gives us flat text with bounding boxes. We re-run a line-clustering pass, detect character height anomalies to identify superscripts and subscripts, then do a separate pixel-level scan for horizontal dark lines — fraction bars — independent of character recognition."

*[Hold up the printed code packet.]*

> "Every line in this packet was written by the three of us. The dev log has timestamped commits for each feature from each author."

## 5:30 – 6:30 | Code Walkthrough (rotating, 20 seconds each)

**Aarav — composePosition (spatial-mapping.js):**
> "This function walks the parent chain from the current AST node back to root, summing 3D offset deltas. A numerator adds (0, +0.6, 0) — up. A superscript adds (+0.4, +0.3, 0) — up and right. The result feeds directly into the HRTF PannerNode's position parameters."

**Person 2 — fftInPlace (fft.js):**
> "Cooley-Tukey radix-2. The outer loop doubles the butterfly length each stage: 2, 4, 8, up to N. The inner loop computes the twiddle factor — a complex rotation — and applies the butterfly: u+v and u-v in place. Bit-reverse permutation at the start ensures correct output order."

**Person 3 — detectVerticalRole (math-cleanup.js):**
> "Each OCR token has a bounding box. We compute the median y-center and height for the line. If a token's y-center is more than 25% of median height above the line, and its own height is less than 80% of median — it's a superscript. Same logic inverted for subscript."

## Q&A Notes (3 minutes)

- The person who **owns the module** being asked about answers **first**.
- Other teammates add only if directly invited.
- If you don't know, say "let me think about that" — never guess under pressure.
- Refer to CODE_DEFENSE.md for anticipated questions per module.

**Most likely judge questions:**
1. "How does HRTF work?" → Head-related transfer functions model how each ear receives a signal from a 3D source. The browser has precomputed HRTF datasets. We set PannerNode positions and the browser does the convolution.
2. "Walk me through the FFT." → Person 2 owns this cold.
3. "What if the camera can't read the equation?" → Graceful fallback: show confidence level, let user retry or use the image upload option. We document the limitation.
4. "Why not just use an existing accessibility app?" → Existing apps (JAWS, NVDA) read equations linearly. No existing free tool provides spatial audio tree navigation or camera-to-audio capture.
